# TT ShadUni 发包与部署指南

## 一、npm 发包

### 1. 构建

```bash
# 完整构建（ESM + CJS + 类型声明）
npm run build:all

# 单独构建 JS
npm run build

# 单独构建类型声明
npm run build:dts
```

构建产物在 `dist/` 目录下：
- `dist/es/` — ESM 格式（preserveModules，按需加载友好）
- `dist/lib/` — CJS 格式
- `dist/types/` — TypeScript 类型声明

### 2. 发布到 npm

```bash
# 方式一：使用 release 脚本（推荐）
# 自动 bump 版本 → 生成 CHANGELOG → git commit + tag
node scripts/release.mjs patch   # 补丁版本 1.0.0 → 1.0.1
node scripts/release.mjs minor   # 次要版本 1.0.0 → 1.1.0
node scripts/release.mjs major   # 主要版本 1.0.0 → 2.0.0

# 推送 tag 到远程
git push --follow-tags

# 发布到 npm（prepublishOnly 会自动执行 build:all）
npm publish

# 方式二：手动发布
npm version patch  # 或 minor / major
npm publish
```

### 3. 用户安装使用

```bash
npm install tt-shaduni
```

在 UniApp 项目中使用：

```vue
<script setup>
import { TtButton, TtIcon } from 'tt-shaduni'
</script>

<template>
  <tt-button type="primary">Click Me</tt-button>
  <tt-icon name="ri-home-line" :size="48" />
</template>
```

如果项目配置了 easycom（UniApp 默认开启），组件会自动注册，无需手动 import。

配合 resolver 按需加载：

```ts
// vite.config.ts
import { TtUIResolver } from 'tt-shaduni/resolver'
import Components from 'unplugin-vue-components/vite'

export default {
  plugins: [
    Components({
      resolvers: [TtUIResolver()],
    }),
  ],
}
```

### 4. 按需加载与 Tree Shaking 说明

**Q: resolver 按需加载是否会在最终打包时去除不用的组件？**

**是的。** 原理：

1. **构建产物是 preserveModules 格式**：每个组件被打包为独立文件（如 `dist/es/components/tt-button/index.mjs`），而非合并成一个大 bundle。

2. **Resolver 自动按需导入**：`TtUIResolver` 配合 `unplugin-vue-components`，只为模板中实际使用的 `<tt-xxx>` 组件生成 import 语句。未使用的组件不会被引入。

3. **Tree Shaking 生效条件**：
   - `package.json` 中配置了 `"sideEffects": ["*.css"]`，告诉打包工具只有 CSS 文件有副作用
   - ESM 格式 + 具名导出，Vite/Webpack 可以静态分析并移除未使用的导出

**三种使用方式的 Tree Shaking 效果：**

| 方式 | 是否 Tree Shake | 说明 |
|------|----------------|------|
| Resolver 按需加载 | ✅ 最优 | 只导入模板中使用的组件 |
| 具名导入 `import { TtButton } from 'tt-shaduni'` | ✅ 有效 | 打包工具可分析未使用的导出 |
| 全量导入 `import * as TtUI from 'tt-shaduni'` | ❌ 无效 | 所有组件都会被打包 |
| easycom 自动注册 | ⚠️ 取决于配置 | UniApp easycom 按需加载，但不走 Vite tree shaking |

## 二、uni_modules 格式发包

uni_modules 是 DCloud 生态的插件格式，可以直接在 HBuilderX 中导入使用。

### 1. 生成 uni_modules 产物

```bash
npm run build:uni_modules
```

产物在 `dist/uni_modules/tt-shaduni/`，包含：
- `components/` — 所有 tt-xxx 组件（不含测试文件）
- `composables/` — 组合式函数
- `styles/` — 样式文件
- `utils/` — 工具函数
- `package.json` — uni_modules 元数据（含 easycom 配置）

### 2. 本地使用

将 `dist/uni_modules/tt-shaduni/` 复制到目标项目的 `uni_modules/` 目录下即可。

### 3. 发布到 DCloud 插件市场

1. 在 [DCloud 插件市场](https://ext.dcloud.net.cn/) 注册账号
2. 用 HBuilderX 打开 `dist/uni_modules/tt-shaduni/`
3. 右键 → 发布到插件市场
4. 填写插件信息、截图、文档链接
5. 提交审核

或使用命令行：
```bash
# 需要先在 HBuilderX 中登录 DCloud 账号
# 然后在 HBuilderX 中操作发布
```

## 三、部署文档到 uniCloud 前端网页托管

uniCloud 提供免费的前端网页托管，适合部署静态文档站。

### 前置条件

1. 注册 [DCloud 开发者账号](https://dev.dcloud.net.cn/)
2. 在 [uniCloud 控制台](https://unicloud.dcloud.net.cn/) 创建服务空间（阿里云或腾讯云）
3. 开通前端网页托管

### 方式一：通过 HBuilderX 部署

1. 先构建文档站：
```bash
npm run docs:playground
```

2. 在 HBuilderX 中：
   - 菜单 → 发行 → 上传网站到服务器
   - 选择 `docs/` 目录
   - 选择目标服务空间
   - 点击上传

### 方式二：通过 CLI 部署

```bash
# 安装 uniCloud CLI
npm install -g @dcloudio/unicloud-cli

# 登录
unicloud login

# 部署 docs 目录到前端网页托管
unicloud hosting deploy --source ./docs --space-id <your-space-id>
```

### 方式三：通过 uniCloud 控制台手动上传

1. 打开 [uniCloud 控制台](https://unicloud.dcloud.net.cn/)
2. 选择服务空间 → 前端网页托管
3. 上传 `docs/` 目录下的所有文件
4. 访问分配的域名即可

### 自定义域名

在 uniCloud 控制台 → 前端网页托管 → 基础配置中：
1. 添加自定义域名
2. 配置 CNAME 解析到 uniCloud 提供的域名
3. 等待 DNS 生效
