# TT-UI 组件库设计文档

> 基于 UniApp + Vue3 + TypeScript 的多端 UI 组件库，采用 shadcn 设计风格

---

## 1. 项目概述

### 1.1 目标

构建一套 **生产级** 的 UniApp Vue3 组件库，核心目标：

- **多端兼容**：一套代码覆盖 H5 / 微信小程序 / 支付宝小程序 / App（iOS/Android）
- **shadcn 风格**：采用 shadcn/ui 的语义化 CSS Variables 主题系统和视觉风格
- **按需引入**：支持 Tree Shaking，未使用的组件不打包
- **主题定制**：通过 CSS Variables 实现运行时主题切换，内置 Light / Dark 模式
- **TypeScript 优先**：完整的类型定义，开箱即用的 IDE 提示

### 1.2 命名

- **包名**：`tt-ui`
- **组件前缀**：`Tt`（如 `TtButton`、`TtInput`）
- **CSS 前缀**：`tt-`（如 `.tt-button`、`.tt-input`）
- **CSS Variable 前缀**：`--tt-`（如 `--tt-primary`）

## 3. 设计原则

### 3.1 shadcn 风格适配

shadcn/ui 的核心理念是 **"不是组件库，而是可复用的组件集合"**。在 UniApp 场景下，我们保留其设计精髓并做适配：

| shadcn/ui 原生 | TT-UI 适配方案 |
|---------------|---------------|
| Tailwind CSS 驱动样式 | SCSS + CSS Variables（UniApp 不完整支持 Tailwind） |
| 复制粘贴到项目 | npm 包 + easycom 自动注册 |
| Radix Primitives 底层 | 自研 UniApp 兼容 Primitives |
| OKLch 色彩空间 | 在 H5 端使用 OKLch，小程序端降级为 HSL |
| React 组件 | Vue3 `<script setup>` + Composition API |

### 3.2 核心设计规范

1. **语义化 Token**：所有颜色使用语义化命名（`--tt-primary`），不直接使用色值
2. **配对模式**：每个表面色搭配前景色（`--tt-card` + `--tt-card-foreground`）
3. **圆角系统**：统一的 `--tt-radius` 变量，衍生 `sm / md / lg / xl` 尺寸
4. **间距系统**：基于 4px 的间距梯度（4, 8, 12, 16, 20, 24, 32, 40, 48）
5. **字体系统**：系统字体栈，尺寸使用 rpx 保证多端一致

### 4.2 主题切换机制

```
用户 → 切换主题 → 修改根节点 class (light/dark)
                  → CSS Variables 自动生效
                  → 所有组件实时响应
```

实现方式：在 `App.vue` 根节点通过 `provide/inject` + `uni.setStorageSync` 持久化主题状态。

### 4.3 自定义主题

用户可通过覆盖 CSS Variables 实现自定义主题：

```css
:root {
  --tt-primary: #6366f1;         /* indigo */
  --tt-primary-foreground: #fff;
  --tt-radius: 16rpx;            /* 更大圆角 */
}
```

### 6.3 easycom 配置

使用者在 `pages.json` 中添加以下配置即可自动注册组件：

```json
{
  "easycom": {
    "autoscan": true,
    "custom": {
      "^tt-(.*)": "tt-ui/packages/components/tt-$1/tt-$1.vue"
    }
  }
}
```

npm 安装方式：

```bash
npm install tt-ui
```

### 6.4 按需引入方案

**方案 A：easycom 自动注册（推荐）**

通过 UniApp easycom 机制，模板中直接使用 `<tt-button>` 即可，无需手动导入。easycom 本身就是按需加载。

**方案 B：手动引入**

```vue
<script setup lang="ts">
import { TtButton } from 'tt-ui'
</script>

<template>
  <TtButton variant="default">按钮</TtButton>
</template>
```

**方案 C：Vite 插件自动导入**

配合 `unplugin-vue-components` 实现自动导入 + Tree Shaking：

```ts
// vite.config.ts
import Components from 'unplugin-vue-components/vite'
import { TtUIResolver } from 'tt-ui/resolver'

export default {
  plugins: [
    Components({
      resolvers: [TtUIResolver()]
    })
  ]
}
```

### 7.1 组件文件结构（以 Button 为例）

```

├── tt-button.vue        # 组件实现
├── props.ts             # Props 定义
├── types.ts             # 类型定义
```

**props.ts**
```typescript
import type { ExtractPropTypes, PropType } from 'vue'

export type ButtonVariant = 'default' | 'destructive' | 'outline' | 'secondary' | 'ghost' | 'link'
export type ButtonSize = 'sm' | 'md' | 'lg' | 'icon'

export const buttonProps = {
  variant: {
    type: String as PropType<ButtonVariant>,
    default: 'default'
  },
  size: {
    type: String as PropType<ButtonSize>,
    default: 'md'
  },
  disabled: {
    type: Boolean,
    default: false
  },
  loading: {
    type: Boolean,
    default: false
  },
  block: {
    type: Boolean,
    default: false
  },
  openType: {
    type: String as PropType<string>,
    default: ''
  }
} as const

export type ButtonProps = ExtractPropTypes<typeof buttonProps>
```

**tt-button.vue**
```vue
<template>
  <button
    class="tt-button"
    :class="rootClass"
    :disabled="disabled || loading"
    :open-type="openType"
    @click="handleClick"
  >
    <tt-loading v-if="loading" size="sm" />
    <slot />
  </button>
</template>

<script setup lang="ts">
import { computed } from 'vue'
import { buttonProps } from './props'

const props = defineProps(buttonProps)
const emit = defineEmits<{ click: [e: Event] }>()

const rootClass = computed(() => [
  `tt-button--${props.variant}`,
  `tt-button--${props.size}`,
  {
    'tt-button--disabled': props.disabled,
    'tt-button--loading': props.loading,
    'tt-button--block': props.block
  }
])

function handleClick(e: Event) {
  if (!props.disabled && !props.loading) {
    emit('click', e)
  }
}
</script>

<style lang="scss">

  }
  &--secondary {

  }
  &--ghost {
    background: transparent;
```

### 7.2 组件 API 设计原则

1. **Props 语义化**：用 `variant` 而非 `type`（与原生属性冲突），用 `size` 统一尺寸命名
2. **事件前缀**：不使用 `on` 前缀，直接 `click`、`change`、`update:modelValue`
3. **双向绑定**：表单组件统一支持 `v-model`，底层使用 `update:modelValue`
4. **插槽命名**：`default`（默认）、`prefix`（前置）、`suffix`（后置）、`header`、`footer`
5. **平台差异**：通过 `conditional compilation`（`#ifdef`）处理，不暴露给使用者

```

│
├─ 通用层（90%+ 代码）
│   └─ Vue3 标准写法，使用 UniApp API
│
├─ H5 适配层
│   └─ #ifdef H5 ... #endif
│
├─ 小程序适配层
│   └─ #ifdef MP-WEIXIN ... #endif
│
└─ App 适配层
    └─ #ifdef APP-PLUS ... #endif
```

## 10. 与 shadcn/ui 的关键差异

| 维度 | shadcn/ui (Web/React) | TT-UI (UniApp/Vue3) |
|------|----------------------|---------------------|
| 分发方式 | CLI 复制到项目 | npm 包 + easycom |
| 样式方案 | Tailwind CSS | SCSS + CSS Variables |
| 底层原语 | Radix Primitives | UniApp 原生组件 |
| 色彩空间 | OKLch | HEX（兼容小程序） |
| 运行环境 | 浏览器 | H5 + 小程序 + App |
| 组件补充 | 纯 UI 组件 | 包含移动端特有组件（Picker、ActionSheet、Tabbar 等） |
| 无障碍 | ARIA 属性 | 平台原生无障碍支持 |
