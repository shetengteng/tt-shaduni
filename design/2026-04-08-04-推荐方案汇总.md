# TT-UI 推荐方案汇总

> 从设计文档、包体积分析、主题切换设计中提取的最终推荐方案
> 已整合评审反馈（v2）

---

## 1. 项目定位

| 维度 | 决策 |
|------|------|
| 包名 | `tt-ui` |
| 组件前缀 | `Tt`（PascalCase）/ `tt-`（kebab-case） |
| CSS Variable 前缀 | `--tt-` |
| 技术栈 | Vue 3 + TypeScript + SCSS + Vite |
| 设计风格 | shadcn/ui（语义化 Token + 配对模式） |
| 目标平台 | H5 / 微信小程序 / 支付宝小程序 / App |

### 1.1 差异化定位

现有 UniApp 组件库（wot-design-uni、TuniaoUI 等）侧重功能完整性，视觉风格偏传统移动端。TT-UI 的差异点：

- **shadcn 视觉语言移植**：将 Web 端最受欢迎的 shadcn 极简风格带入 UniApp 多端场景
- **Token-First 设计**：组件库的核心产物不仅是组件，更是一套可复用的语义化设计 Token 体系
- **轻量 MVP 起步**：不追求 80+ 组件的全覆盖，优先打通 10 个核心组件的多端验证

## 3. 按需加载方案

### 3.1 推荐方式：easycom + Vite 插件双轨

**安装方式 A：uni_modules（HBuilderX 用户）**

直接拷贝到 `uni_modules/tt-ui/`，easycom 自动生效，零配置按需引入。

**安装方式 B：npm + Vite 插件（CLI 用户，推荐）**

```bash
npm install tt-ui
```

```typescript
// vite.config.ts
import Components from '@uni-helper/vite-plugin-uni-components'
import { TtUIResolver } from 'tt-ui/resolver'

export default defineConfig({
  plugins: [
    Components({ resolvers: [TtUIResolver()] }),
    uni()
  ]
})
```

### 3.3 构建配置

```typescript

import { defineConfig } from 'vite'

import dts from 'vite-plugin-dts'

    rollupOptions: {

      output: {
        preserveModules: true,

        entryFileNames: '[name].mjs',

      }
    },
    cssCodeSplit: true
  }

```

> **注意**：必须包含 `@vitejs/plugin-vue` 以编译 `.vue` SFC 文件，`vite-plugin-dts` 用于自动生成 `.d.ts` 类型声明。

### 3.4 Resolver 组件依赖处理

当组件 A 内部使用组件 B 时（如 Button 使用 Loading），Resolver 需要自动引入依赖链：

```typescript
// resolver.ts 中的依赖映射
const componentDeps: Record<string, string[]> = {
  'tt-button': ['tt-loading'],
  'tt-dialog': ['tt-popup', 'tt-button'],
  'tt-form': ['tt-input'],
}

export function TtUIResolver(options: TtUIResolverOptions = {}) {
  return {
    type: 'component' as const,
    resolve: (name: string) => {
      if (!name.startsWith('Tt')) return

      const kebabName = toKebab(name)
      const deps = componentDeps[kebabName] || []
      const sideEffects = [
        `tt-ui/es/${kebabName}/style.css`,
        ...deps.map(d => `tt-ui/es/${d}/style.css`)
      ]

      return {
        name: 'default',
        from: `tt-ui/es/${kebabName}/index.mjs`,
        sideEffects
      }
    }
  }
}
```

### 3.5 组件独立性要求

- 样式写在 `.vue` 的 `<style>` 块中，**不 `@import` 外部 SCSS**
- 工具函数按文件导入：`from '../utils/shared'` 而非 `from '../utils'`
- 组件间允许 import 其他 TT-UI 组件（如 Button 引入 Loading），但需在 Resolver 依赖表中注册

### 4.1 架构：三层分离

```
Layer 3: 微信 DarkMode（处理导航栏、TabBar 等系统 UI）
Layer 2: ConfigProvider + CSS Variables（处理所有组件样式）
Layer 1: use-theme composable（状态管理 + 持久化 + 跨页同步）
```

### 4.3 Token 命名转换规则

TS 中使用 camelCase，注入 CSS 时自动转为 `--tt-` 前缀的 kebab-case：

| TS Key | CSS Variable |
|--------|-------------|
| `primary` | `--tt-primary` |
| `primaryForeground` | `--tt-primary-foreground` |
| `mutedForeground` | `--tt-muted-foreground` |

转换函数：

```typescript
function tokenToVar(key: string): string {
  const kebab = key.replace(/([A-Z])/g, '-$1').toLowerCase()
  return `--tt-${kebab}`
}
```

### 4.4 主题注入方式：ConfigProvider `:style` 内联

**不使用** `:root` / `.dark` class（小程序不支持）。
**使用** ConfigProvider 组件的 `:style` 属性内联注入 CSS Variables。

这是在所有平台（H5 / 小程序 / App）都可靠继承的方式。`:style` 设置的 CSS Variables 作为 DOM 属性向下继承，不受小程序组件样式隔离限制。

**前提条件**：所有 TT-UI 组件 **不得** 设置 `options.styleIsolation` 为 `'isolated'`。

**initTheme 调用时机**：在 ConfigProvider 的 `setup()` 中自动调用 `initTheme()`，确保首次渲染前主题状态已从 Storage 恢复，避免主题闪烁。

```typescript
// ConfigProvider setup 中
import { useTheme } from '../../composables/use-theme'
const { resolvedTheme, initTheme } = useTheme()
initTheme()  // 首次渲染前恢复主题
```

### 4.5 嵌套 ConfigProvider 的 Token 合并

嵌套时采用 **diff merge** 策略 — 内层仅注入与父级不同的变量：

```
外层 ConfigProvider (theme="light", 注入 22 个 light token)
  └─ 内层 ConfigProvider (theme-vars="{ primary: '#6366f1' }")
       仅注入 --tt-primary: #6366f1（1 个变量）
       其余变量从外层继承
```

实现方式：内层 ConfigProvider 通过 `inject` 获取父级 Token，计算 diff 后仅输出差异部分。

### 4.6 用户使用方式

```vue
<!-- 每个页面包裹一次 -->
<template>
  <tt-config-provider>
    <view class="page">
      <tt-button>按钮</tt-button>
    </view>
  </tt-config-provider>
</template>
```

**可选进阶**：配合 `@uni-ku/root` 实现全局一次性配置。

### 4.7 自定义主题

```vue
<tt-config-provider :theme-vars="{ primary: '#6366f1', primaryForeground: '#fff' }">
  <slot />
</tt-config-provider>
```

## 5. 组件开发规范

### 5.1 文件结构

```
tt-button/
├── tt-button.vue    # 模板 + 脚本 + 样式（自包含）
├── props.ts         # Props 类型定义
└── index.ts         # 导出入口
```

### 5.2 组件模板

```vue
<template>
  <button class="tt-button" :class="rootClass" :style="rootStyle">
    <slot />
  </button>
</template>

<script setup lang="ts">
import { computed } from 'vue'
import { buttonProps } from './props'

const props = defineProps(buttonProps)
const emit = defineEmits<{ click: [e: Event] }>()
</script>

<style lang="scss">
.tt-button {
  background: var(--tt-primary, #171717);
  color: var(--tt-primary-foreground, #fafafa);
  border-radius: var(--tt-radius, 6px);
}
</style>
```

### 5.3 API 设计约定

| 约定 | 说明 |
|------|------|
| `variant` | 组件变体（不用 `type`，避免与原生冲突） |
| `size` | 统一尺寸：`sm` / `md` / `lg` |
| `v-model` | 表单组件统一支持，底层 `update:modelValue` |
| 插槽 | `default` / `prefix` / `suffix` / `header` / `footer` |
| 平台差异 | `#ifdef` 条件编译，不暴露给用户 |
| CSS fallback | 所有 `var()` 引用必须提供降级值：`var(--tt-xxx, fallback)` |

### 5.4 组件硬性约束

1. **不得设置 `styleIsolation: 'isolated'`** — 否则 ConfigProvider 的 CSS Variables 无法穿透
2. **样式自包含** — 所有样式写在 `.vue` 的 `<style>` 块中，不 `@import` 外部 SCSS
3. **CSS Variable 必须带 fallback** — 确保未被 ConfigProvider 包裹时组件仍可用
4. **基础 ARIA 属性** — MVP 组件需提供 `role`、`aria-disabled`、`aria-label` 等基础无障碍属性

## 8. 关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| Token 存储格式 | TS 对象（非 SCSS 文件） | 运行时通过 JS 注入，避免 SCSS 编译膨胀 |
| Token 单位 | px（非 rpx） | CSS Variables 中 rpx 在小程序不转换 |
| Token 命名 | camelCase → `--tt-kebab-case` | TS 开发友好，CSS 端自动转换 |
| 主题注入方式 | ConfigProvider `:style` | 跨全平台可靠继承，不受样式隔离限制 |
| 嵌套主题合并 | diff merge（仅注入差异） | 避免内层全量覆盖外层 Token |
| initTheme 时机 | ConfigProvider `setup()` 中自动调用 | 首次渲染前恢复持久化主题，避免闪烁 |
| 分发渠道 | npm + uni_modules 双轨 | 覆盖 CLI 和 HBuilderX 两类用户 |
| 构建产物 | ESM (preserveModules) + Vue plugin + dts | Tree Shaking 到单组件级别，含类型声明 |
| sideEffects 声明 | 仅 CSS 文件 | 让打包工具安全移除未用 JS |
| 组件样式方案 | 内聚在 .vue `<style>` + CSS fallback | 避免外部 @import 膨胀，未包裹时仍可用 |
| 组件依赖链 | Resolver 维护 componentDeps 映射 | 按需引入时自动补充子组件 CSS |
| styleIsolation | 禁止 `isolated`，允许默认 `apply-shared` | 确保 CSS Variables 从 ConfigProvider 穿透 |

## 9. Token 稳定性约定

| 阶段 | 规则 |
|------|------|
| `0.x` | Token 名称可能变更，在 CHANGELOG 中标注 |
| `1.0` 起 | Token 名称变更视为 breaking change，必须升 major 版本 |

Token 名称（如 `--tt-primary`、`--tt-primary-foreground`）本质上是公开 API，用户可能通过 CSS 覆盖或 `theme-vars` 引用。变更会导致所有自定义主题失效。
